# REQ-011 需求规格

## FR-011-1: Supersedes 排除
**场景**: 记忆 A 的 `supersedes` 字段包含记忆 B 的 ID
- 当 A 和 B 同时出现在 recall 结果中时，B 应被排除
- 当只有 B 出现（A 不在结果集中）时，B 保持不变
- 当 A 的 supersedes 包含多个 ID 时，所有被替代的记忆都应被排除

## FR-011-2: 质量加权排序
**场景**: 两条记忆文本相关性相近，但质量不同
- 高 confidence (1.0) 的记忆应排在低 confidence (0.1) 的记忆之前
- 高 importance (1.0) 的记忆应排在低 importance (0.1) 的记忆之前
- 质量乘数应为乘法因子（非加法），确保零质量记忆得零分

## FR-011-3: Veracity 评分
**场景**: 不同来源的记忆有不同的可信度
- `stated` (用户明确陈述) → 1.0
- `tool` (机器生成但可验证) → 0.9
- `inferred` (推断得出) → 0.5
- `imported` (导入，来源不明) → 0.5
- `unknown` (未知) → 0.3

## FR-011-4: MMR 质量感知
**场景**: MMR rerank 在多样性与相关性之间权衡
- MMR 的 relevance 计算应包含质量乘数
- 高质量记忆在多样性选择中应有优势

## 非功能需求
- 所有新行为必须有对应的单元测试
- 不改变现有 CLI 接口（recall 命令参数不变）
- 性能影响可忽略（supersedes 后处理在 ≤20 条结果上运行）
